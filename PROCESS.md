# PROCESS

本文件只记录可复用的反回归原则和工程判断，不记录单次流水账。项目状态、当前阶段和下一步见 `PROJECT_STATUS.json`；发布 manifest、截图、测试输出保留在 git 历史、release 目录或对应 artifact 中。

## 维护规则

- **只写会二次踩坑的经验**：如果一条记录只是“做了什么、提交号、截图、测试通过”，不要写进这里；只有当它抽象出未来应遵守的判断、边界、排查顺序或失败模式时才写。
- **每条记录必须可执行**：优先使用“触发场景 → 原则 → 必做动作/禁止动作”的写法。避免“已修复/已发布/已验证”这类完成态描述。
- **不要把短期事实写成长期原则**：股票代码、run id、commit、截图路径、单次收益、一次性回填结果不进 PROCESS；这些属于状态文件或研究 artifact。
- **保留边界，不保留噪声**：当某次事故只贡献一个新边界条件，把边界条件归入对应主题；删除复盘中的路径、命令输出和重复验证。
- **新增条目前先查重**：同类原则已存在时，修改原条目而不是追加日期流水。
- **状态与原则分离**：`PROJECT_STATUS.json` 记录“现在到哪了”；`DECISIONS.md` 记录“为什么选择这个方向”；`PROCESS.md` 记录“以后遇到同类问题怎么不再错”。

## 完成定义

- **live-facing 任务不能停在 repo**：用户可见修复的完成链路是 repo 改动 → 本地测试/构建 → 发布到 runtime → 真实 served 页面或 API 验收。缺发布或缺真实验收时，只能说“代码已改”，不能说完成。
- **发布必须对应干净源提交**：发布前确保工作树可解释；若文档/状态在发布后又变化，需要再次发布或明确这些变化不影响 runtime。不要让 runtime manifest 长期落后于最终收口提交。
- **浏览器验收要验证真实问题点**：不要只验证页面能打开。移动端问题用移动视口检查布局指标；数据展示问题用 served 页面文本/API 同时确认；canonical 问题优先核对认证、隧道和远端路由。
- **发布刷新不能掩盖修复边界**：代码展示层修复可用 `ASHARE_PUBLISH_REFRESH_MODE=skip`，但数据链路或 run 内容修复必须补相应 runtime DB 投影/刷新，并验证实际接口返回。
- **长任务不留悬挂进程**：批量回放、浏览器自动化、短投验证、刷新脚本结束后检查并清理相关进程和 lock；不要误杀用户普通浏览器。

## AI 开发流程

- **全局协议先于并行模块**：多 agent 设计或实现前，先由主进程产出术语、事件、artifact、接口和成熟度 registry；依赖该 registry 的模块任务不能与 registry 任务同轮并行，否则容易出现模块自造事件、接口漂移和后续拼接失败。
- **子进程输入要压缩成 Context Pack**：不要让每个子进程重复读取长状态文件和全部历史合同。主进程先冻结目标、非目标、owned files、registry allowlist、成熟度限制、必读文件和禁止动作，再派发短输入；需要长文档时只给路径和具体读取范围。
- **重跑必须修复失败面而不是补解释**：评审发现未注册接口、成熟度越界、开放问题未分类或子进程越权时，进入下一轮重跑，把评审结论升级为硬约束；不能只在主进程总结里解释为什么问题“可以接受”。
- **设计阶段也要有可机检门禁**：Markdown 合同可以作为早期草案，但至少要提供 registry appendix 或 allowlist，支持脚本检查未注册事件、artifact、interface 和废弃 id；进入代码实现前再决定是否升级为 JSON Schema、DB registry 或代码生成清单。

## 运行态与发布

- **runtime DB 是 live 真值源**：repo 本地库、样本库、output artifact 只能用于复现和研究；对用户解释当前页面时以 runtime DB/API 为准。
- **artifact 写入保护要识别项目级 `.git`**：开发仓里的 `data/artifacts` 仍属于源码污染风险，必须拒绝刷新脚本直接写入；同步后的 runtime 目录没有项目级 `.git`，其中的 `data/artifacts` 是 live 数据目录，不能被同一保护误判为 source checkout。
- **canonical 不可用先分层排查**：先查 localhost 5173/8000 健康，再查 canonical 是否 302 到登录页，再查 tunnel/remote port ownership；不要先误判为代码失效。
- **canonical stale 多半是路由或隧道问题**：页面旧、接口新、资源健康时，优先检查 tunnel、远端端口和缓存态，再决定是否重新发布。
- **LaunchAgent 要区分任务和服务**：定时任务用 `RunAtLoad` + `StartCalendarInterval` 或 `StartInterval`；服务进程用 `RunAtLoad` + `KeepAlive`。需要精确时钟触发时不能只靠 `StartInterval=300`；关键时点应配置显式 calendar trigger，并让任务用 slot state 保证多次唤醒不重复写入。
- **SSH 隧道要自愈且能清旧占用**：重连失败先查远端旧 sshd 是否占端口；清理脚本改动后跑语法检查；隧道进程遇错应退避重试。
- **SQLite 写锁要按工作流避让**：批量短投验证、历史回放、盘后刷新和页面验收不要无脑并发；遇到等待时先判断锁持有者，不要把慢写误判为 Python 崩溃。
- **维护型短投补算不能挂在前台热路径**：recent validation catch-up、failed-round retry、长窗口回测和行业归因补数都属于维护任务；默认不要接在 live daily shortpick slot 后继续占 SQLite 写锁。需要补跑时用显式开关或维护窗口，并在前后检查前端小接口健康。
- **前端统计读数要走投影表**：首屏统计、聚合历史结论、运营摘要和模型反馈趋势应由定时/触发式任务写入 `frontend_projections`，API 只读小 payload。页面请求不得临时扫大表、读整包 artifact、跑统计或写库。
- **大 universe 回测必须先扁平化数据**：3000 只股票级别的历史回测不要在循环里反复构造 ORM 对象或重复扫描全市场基准；先把日线、特征、候选池和基准路径变成可复用的扁平缓存，再做小网格或精确确认。

## 策略与研究

- **冻结策略不等于跳过治理**：池子大小、TopK、阈值、惩罚系数、退出窗口、入场口径和对照版本都必须进入 policy config，并由代码投影使用；policy audit 要覆盖新策略域。
- **冻结主线只能按原证据口径前向观察**：已冻结的主策略不要因为一两天纸面表现改参数。新想法用并行对照组验证，不能直接替换主线。
- **入场口径变更是新研究，不是小修**：次日收盘、次日开盘、涨停可成交假设会改变历史证据；必须并行记录 entry price source，并让页面展示清楚。
- **盘中限时对照不要跑完整研究链**：如果用户要求固定时点前出结果，先判断耗时上界；能准点的只能是确定性规则 + 快照数据，LLM daily-analysis 和大规模补数继续留在盘后批次。
- **涨停可执行性要保守标注**：一字涨停不能假设买入；开盘入场研究遇到接近涨停开盘也不能假设成交。普通收盘涨停如果日线无法证明无成交，应在口径说明里暴露不确定性。
- **账户可买范围先于选股优先级**：严格纸面对照和可执行研究必须先过 `new_retail_cash_account` 过滤；创业板、科创板、北交所、ST/退市风险标的可保留为研究样本，但不能进入新开户普通现金账户严格交易对照。
- **对照组要回答单一问题**：前三等权回答单票依赖，金叉回答趋势确认，非涨停追高回答可执行性，开盘版回答入场时点。不要把多个假设混进一条对照。
- **候选验证与资金曲线不能混排**：逐候选短窗口均值回答“选股池平均质量”；组合资金曲线回答“按资金规则执行后的账户路径”。展示和结论必须分口径。
- **指数/基准缺失不能显示成 0**：缺三大指数或同行基准时显示缺失/不可用，用等权市场基准或明确的 fallback；不要让空序列在 UI 上像真实 0%。
- **小样本均值不能包装成稳定策略**：如果收益来自少数强尾部或中位数为负，应明确“观察性研究”，不能提升为生产级证明。
- **优化候选必须先校准基线**：任何更快的回测器、向量化筛选器或近似模拟器，必须先复现当前已确认策略在 canonical artifact 中的核心结果，再用来替换冻结线。不能用一个尚未校准的快工具证明“新策略更好”。
- **冻结可以分级但不能混名**：当一条规则是“当前最佳已确认纸面候选”但未通过生产门禁时，页面和结论都要保留这个降级语义；不要把纸面观察、研究候选、生产证明三个等级混成一个“冻结策略”。
- **盘中 SLA 要拆成外部拉数和本地计算**：14点这类固定时点任务必须分别记录行情源耗时、粗筛耗时和选股耗时。本地计算达标不代表整条链路达标，实时行情源超时要按独立风险治理。

## Short Pick Lab

- **纸面跟踪 API 必须只读**：打开试验田、纸面跟踪或历史回放不能触发行情同步、模型调用、DB 写入或全量市场研究。
- **历史回放页面只能读预计算统计**：feedback、策略收口和组合回测摘要必须由离线脚本或已完成 artifact 生成；页面请求不能临时扫 validation 表或跑 market-factor study。
- **当前/下轮语义必须看 signal date 和 entry date**：已入场记录、下轮选择、当前交易中选择要分开展示。不要用最新 run 覆盖已入场 ledger。
- **历史 seed 要显式写 signal/entry date**：冻结前已有人工盯盘名单时，回填必须带原信号日和实际入场日，避免页面显示“没有纸面追踪”。
- **LLM 纸面对照只能从可买池里固定一只**：全量 LLM 候选保留研究用途；进入严格纸面对照的唯一候选必须先过账户可交易性过滤，再按固定排序提前标记。
- **市场因子 overlay 删除重建要局部化**：重新投影某个 run 时只删除该 run 的 market-factor overlay 和关联验证，不影响 LLM 原始候选和其他运行记录。
- **无 live run 也要能看历史回放**：试验田 tab 的可见性不能只挂在 latest live run 上；只有 replay 数据时也应显示历史回放。
- **短投列表接口要轻量**：run 列表不嵌 full candidates、source packet、raw answer；大 payload 走下钻接口，避免打开试验田时解析数 MB JSON。
- **历史回放聚合接口要投影后再下发**：长窗口策略证据可以来自重 artifact，但 `/shortpick-lab/replay-feedback` 只应返回首屏和当前下钻需要的 rows；全量 period/regime/time-slice 明细留在 artifact 或专门下钻接口里。
- **历史分析结论只做 projection**：首屏决策读数、可执行性漏斗、入场矩阵必须由已完成 artifact 或只读 ledger 投影；缺字段显示待补原因，不在前端拼结论，也不在页面请求里补跑研究任务。
- **长窗口策略证据要和 LLM replay 分名**：当 LLM 历史 replay 时间窗过短时，可以用 full-window staged portfolio artifact 扩充确定性策略族的跨时段和分行情证据，但页面必须明确它回答的是组合路径稳定性，不等同于扩充了 LLM 自由选股样本。
- **非 LLM 高级分析也要声明数据粒度**：组合级置信区间、稳定性和归因可以从 staged portfolio 月度/季度/行情桶产物投影；股票/行业归因必须等全量逐笔 artifact，不能用 `trades_sample` 外推。
- **纸面退出结果要默认浮出**：机械 5 日这类已完成退出不能只藏在搜索和翻页里；试验田应有退出数量、等待窗口、最新退出和退出状态筛选。落实见 `8ef154e Surface shortpick paper tracking exits`。

## 历史回放与证据隔离

- **历史回放必须 point-in-time**：prompt 只能包含 sealed source packet 和当日可交易 universe；不得调用现时搜索、现时行情补数或未来 exit bars。
- **诊断代理不能冒充真实 LLM**：proxy 只能显式标为 diagnostic；真实 LLM 失败就记录失败 round，不自动生成可混入正式统计的 proxy 候选。
- **来源引用失败要 fail closed**：候选引用 rejected source、packet 外 source id、cutoff 后来源或未验证时间来源时，应保留原始引用并触发 leakage audit failure，不要静默替换成看似合法来源。
- **unsupported claim 不等于未来泄漏**：严格来源样本和可交易验证样本要拆开。来源证据不足可以退出 strict 样本，但只要无硬泄漏且账户可买，可进入可交易口径观察。
- **历史回放首屏先回答研究问题**：先展示覆盖日期、正式样本、周期表现、对照组和统计门槛；packet/source/泄漏审计放下钻区。
- **历史回放样本主体应是 candidates**：不要只依赖单个 primary pick；模型输出预算和 repair 逻辑要支持候选列表，避免样本数不足。

## 多账号与权限

- **先分清真串号和展示误导**：多账号问题先用 API 验证账号态和数据归属，再判断是后端隔离失败还是前端把全局候选池混进账号工作台。
- **`act_as` 不跨刷新持久化**：root 代看是单页临时状态；刷新或重开标签默认回 actor 自己空间，避免用户误以为持仓消失。
- **member 初始化顺序不能复用 root**：先取 `/auth/context`；root 可走 runtime settings，member 应走 overview/只读入口，避免 settings 403 白屏。
- **个人模型 Key 必须按账号隔离**：人工研究和追问是所有账号的工作流，不应因为 member 不是 root 而隐藏；但外部模型 API key 必须按 `target_login` 读写、默认和 failover，root 代看时也只能维护被代看账号自己的 Key。全站数据源 token 仍按运维配置处理，不混入个人模型配置。
- **关注池隔离不等于分析断流**：账号 follow 分开存；symbol 级分析资格按所有账号 active follows 并集判断。只有最后一个 follow 消失，才停止全局 tracking。
- **空成员空间不能启动空模拟盘**：member 首登可以有 draft session，但 `can_start` 和 API 都必须拒绝空 watch symbols。

## 前端与移动端

- **移动端不要压缩 PC 工作台**：`App.tsx` 保持数据和 handler 入口，手机端使用 `components/mobile/*` 独立 shell；宽表和 PC tab 不应直接塞进移动端。
- **移动端交互要有明确 affordance**：只读行不显示右箭头；二态偏好用 Switch；多选/模型选择才进入二级菜单。
- **破坏性滑动操作只在展开态显红**：闭合态不要透出红色；展开态处理相邻圆角；释放后吞掉下一次 click，防误开详情。
- **底部导航内容按整体居中**：按钮不要用容易受字体和安全区影响的两行 grid；图标和文字作为一个整体垂直居中，并固定 line-height/尺寸。
- **加载态和空态要分清**：接口未完成时显示加载中/骨架；确认无数据才显示暂无。不要用 `-` 让用户分不清是空还是慢。
- **页面应先回答用户问题**：历史回放、策略收口、复盘页首屏放结论性统计和操作入口；技术审计、source packet、raw payload 放下钻。
- **用户可见文案禁止内部指令味**：避免 `must/builtin/verified` 这类实现词；用“探索结果 / 默认模型 / 短窗口候选统计 / 长样本账户路径回测”等业务词。

## 数据与分析流水线

- **外部数据 fail-open**：AKShare、公告、实时信息流等外部接口可能超时、空表或异常；刷新 pipeline 中这类失败应降级为空数据/告警，不应中断 intraday sync、simulation step 或其他主流程。
- **事件分析接入不能阻断主刷新**：新增事件检测、LLM 分析、资料抓取时必须包裹异常，避免一个新模块拖垮盘后全流程。
- **simulation 刷新顺序固定**：不能只 restart session；需要 `restart → step → rebuild`。持续运行用交易时段后台 tick + 单次 anchored step，不复用研究刷新链路。
- **Phase 5 artifact 查找要容忍 legacy 字段漂移**：payload 中 `backtest_artifact_id` 等旧字段可能不稳定，优先解析真实存在 artifact。
- **盘后 freshness 文案用快照时间**：用户可见层显示“截至 MM/DD HH:MM”，不要暴露原始秒数或内部 freshness 算法。

## API、脚本与工具陷阱

- **API 错误响应统一用 `detail`**：前端只提取 `payload.detail` 展示中文错误；返回 `error` 会退化成裸 HTTP 状态码。
- **Prompt 文案改动要同步断言**：用户可见 prompt 或 follow-up copy 改词后，跑相关 view/prompt 测试，避免断言继续锁旧词。
- **Pydantic 前向引用不要藏在 `TYPE_CHECKING`**：Pydantic v2 用模块命名空间解析注解；循环依赖时在模块加载完成后注入类型并 `model_rebuild()`。
- **bash 命令拼接不要 `.join('; ')`**：多余分号会制造 `do;`、`then;` 语法错误。生成脚本后跑 `bash -n`。
- **自动化浏览器清理要精准**：清理 Playwright daemon 和临时 profile，不要 `pkill Chrome` 误伤用户会话。
- **文件批量替换要保留行结构**：用脚本改文件时新内容必须按行插入，避免把函数体压成一行；复杂编辑优先用小 patch 或格式化工具验证。
